criteria: Use for cross-project status reporting on an initiative.

---

<querying>
- Pull prior updates to match tone and set reporting window.
- Can include sub-initiative updates and/or related project updates when querying.
</querying>

<drafting>
- Unless explicitly asked to post immediately, draft first for review.
- Lead with the one or two cross-cutting highlights (wins or risks).
- Summarize meaningful movement across sub-initiatives/projects (only what changed recently).
- Call out key dependencies/risks and what's needed (only if known).
- Keep language plain; avoid PM artifact-driven narration.
</drafting>

<health>
- onTrack / atRisk / offTrack — same logic as project updates, but applied across the portfolio.
- Set based on evidence.
</health>

<editing>
- Only update what's requested; author/admin constraints may apply.
</editing>
